# Agent instructions for ogra

## Development environment

This project uses `devenv.nix` for its development environment. All commands should be run
from within the devenv shell. See the user-level rules for how to enter the shell.

Key devenv scripts:

- `backend` — runs the Go backend server (`go run ./cmd/server`)
- `frontend` — runs the Vite dev server for the React frontend
- `testbed` — manages E2E test namespaces (`testbed setup <fixture>`, `testbed teardown <ns>`)
- `ui-test` — runs Playwright UI tests with the Nix-managed Chromium
- `gen-api-types` — regenerates Go and TypeScript types from Argo Workflows CRDs

## Devenv processes

This project defines two devenv processes: `backend` (Go server) and `frontend`
(Vite dev server). They are started together with `devenv up` or `devenv processes start`.

### Do not orphan the process supervisor

The devenv process manager runs a `supervisord`-style daemon that owns all child
processes. If the daemon loses its control socket or its parent shell exits
unexpectedly, the child processes (Go backend, Vite) keep running invisibly in
the background. This causes:

- **Port conflicts** — the next `devenv up` fails because ports are already bound.
- **Stale servers** — code changes are not picked up because the old process is
  still serving.
- **Unkillable children** — manually killing a child (e.g. `kill <vite-pid>`) just
  triggers an immediate restart by the still-running supervisor.

### Rules for agents

1. **Never run `devenv up` or `devenv processes start` in an ephemeral or nested
   shell.** If your shell session is cancelled or times out, the supervisor
   daemon and all its children will be orphaned.
2. **Run `devenv up` in the foreground of a persistent, backgrounded terminal
   session.** Background the session itself (e.g. via a background task with a
   long timeout), not the process manager. When the session is killed, signals
   propagate to the entire process group and all children terminate cleanly.
3. **Always shut down with `devenv processes down`** from the same shell session
   that started the processes. Do not rely on killing individual child PIDs.
4. **If processes are already orphaned**, kill the supervisor first, then mop up
   children:

   ```bash
   pkill -f "daemon-processes"
   pkill -f "node.*vite" || true
   lsof -ti :8082 | xargs kill || true
   ```

5. **Use `devenv test` for one-shot validation.** It starts processes, runs
   tests, and tears everything down automatically — no orphan risk.

## Pre-commit hooks

The following hooks run on every commit via `devenv:git-hooks:run`:

- `nixfmt` — Nix formatter
- `markdownlint` — markdown linting
- `shellcheck` — shell script analysis
- `gofmt` — Go formatter
- `golangci-lint` — Go linter suite (config in `.golangci.yml`)

Do not bypass hooks with `git commit --no-verify`.

To run hooks manually against specific files:

```bash
prek run .:golangci-lint --files <file>
prek run .:markdownlint --files <file>
```

## Generated code

### `internal/api/types.go`

This file is generated by `scripts/gen-api-types.py` from the Argo Workflows CRDs
in `api/crds/` and the OpenAPI schema in `api/openapi.json`. It carries a
`// Code generated by gen-api-types. DO NOT EDIT.` header and is excluded from
`golangci-lint` via `.golangci.yml`.

Do not hand-edit this file. To regenerate:

```bash
gen-api-types
```

### `frontend/src/types/generated.ts`

TypeScript types are also generated by `gen-api-types`. Same rule applies — do not hand-edit.

## Go conventions

- E2E test files use the `//go:build e2e` build tag. The `.golangci.yml` includes
  `build-tags: [e2e]` so the linter picks them up.
- Use `exec.CommandContext(context.Background(), ...)` instead of `exec.Command(...)`.
- Use `errors.As` instead of direct type assertions on errors.
- Use safe type assertions with the `, ok` pattern; the `forcetypeassert` linter
  rejects unchecked assertions.
- Close response bodies with `defer func() { _ = resp.Body.Close() }()` to satisfy
  `errcheck`.

## Frontend conventions

- The frontend lives in `frontend/` and uses Vite + React + TypeScript.
- Run `npm install` from `frontend/` before first use.
- The Vite dev server proxies `/api` requests to the Go backend (configured in
  `frontend/vite.config.ts`).

## Testing

### API E2E tests

API tests require a live Kubernetes cluster with Argo Workflows installed. The `testbed`
script provisions a temporary namespace with fixture resources:

```bash
go test -tags e2e ./tests/e2e/api/ -v
```

### UI E2E tests

Playwright tests require both the backend and frontend running. Use the devenv processes
or start them manually, then:

```bash
ui-test
```
